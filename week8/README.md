# Developer Control Center - Multi-Stack Web Application

A feature-complete web application for managing **notes** and **action items** built in **three distinct technology stacks** to demonstrate modern full-stack development practices.

## 📋 Project Overview

The Developer Control Center is a personal productivity dashboard that allows users to:
- ✅ Create, read, update, and delete notes (with title and content)
- ✅ Create, read, update, and delete action items (with description and completion status)
- ✅ Toggle task completion with an interactive checkbox
- ✅ Persist all data to a database
- ✅ Enjoy a modern, responsive dark-mode UI

**Minimum Functional Scope:**
- Users can perform full CRUD operations on two primary resources (notes and action items)
- Persistent storage via database (PostgreSQL/SQLite depending on stack)
- Input validation with user-friendly error messages
- Clean, intuitive UI that surfaces all main workflows
- End-to-end functionality (frontend + backend + database)

## 💻 API Endpoints (Version 1)

All endpoints are available at `http://localhost:3000/api/`

### Notes API
| Method | Endpoint | Purpose |
|--------|----------|---------|
| GET | `/notes` | Get all notes |
| POST | `/notes` | Create a new note |
| GET | `/notes/[id]` | Get a specific note |
| PUT | `/notes/[id]` | Update a note |
| DELETE | `/notes/[id]` | Delete a note |

### Action Items API
| Method | Endpoint | Purpose |
|--------|----------|---------|
| GET | `/action-items` | Get all action items |
| POST | `/action-items` | Create new action item |
| GET | `/action-items/[id]` | Get specific item |
| PUT | `/action-items/[id]` | Update item (description or completion status) |
| DELETE | `/action-items/[id]` | Delete item |

## ✅ Validation Rules

All versions implement the following validation:

### Notes
- **Title:** Required, cannot be empty or whitespace-only
- **Content:** Required, cannot be empty or whitespace-only

### Action Items
- **Description:** Required, cannot be empty or whitespace-only
- **Completed:** Boolean, optional (defaults to false)

All empty submissions display an error message without submitting to the database.

## 🐛 Troubleshooting

### Version 1 (Next.js)

**"DATABASE_URL not found" error**
- Ensure `.env` file exists in the root with `DATABASE_URL="file:./dev.db"`
- Run: `npx prisma db push --skip-generate`

**Port 3000 already in use**
- Run on a different port: `npm run dev -- -p 3001`

**Dependencies not installing**
- Clear cache: `npm cache clean --force`
- Delete `node_modules` and `package-lock.json`
- Reinstall: `npm install`

**Database file permission errors**
- Ensure write permissions in the project directory
- On Windows, check folder properties > Security
